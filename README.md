# FullStack Production Engineering Skill
> 一套面向 AI 编码的企业级全栈生产规范，让 AI 输出的代码从 Demo 直接达到可上线标准。

---

## ✨ 为什么用它？
市面上的通用 AI 编码，最大的问题就是：**只实现功能，不考虑规范、边界、性能和稳定性**。
这份 Skill 就是为了解决这些问题，给 AI 戴上「生产级编码的紧箍咒」。

-   **✅ 全栈统一编码规范**
    覆盖前端、后端、数据库、工程化全链路，定义了一套可落地的企业级标准。从变量声明、数据处理到组件生命周期，每一条规则都经过线上项目验证。

-   **✅ 根治线上高频 Bug**
    内置大量针对线上坑的强制规则：
    -   前端：Vue3 响应式失效、KeepAlive 内存泄漏、异步静默失败、空值异常
    -   后端：日志脱敏、错误栈泄露、参数校验缺失、请求体攻击
    -   SQL：全表扫描、锁表、`SELECT *` 低效写法
    让 AI 写出来的代码自带「避坑体质」。

-   **✅ 工程化与部署标准约束**
    不只是代码层面，还包含打包配置、静态资源缓存、Nginx/PM2 生产部署的最佳实践，从开发到上线全链路规范。

-   **✅ 智能场景隔离，零 Token 浪费**
    内置精准命中字段，写前端只触发前端规则，后端/SQL 规则自动休眠。既保证了全栈覆盖，又避免了冗余校验和规则串扰。

-   **✅ 自带代码自检与修复能力**
    内置生产级自检流程，AI 写完代码后可一键自检，自动修复内存泄漏、异步逻辑、数据安全等隐性问题。

## 🎯 核心规范亮点
-   **通用强制规则**：杜绝 Demo 代码、硬编码、静默失败，强制资源生命周期闭环。
-   **JS/TS 编码规范**：基础语法、数据处理、异步编程、内存管理的全链路约束。
-   **Vue3 专属规范**：响应式、渲染性能、KeepAlive 组件的生产级最佳实践。
-   **Node 后端规范**：分层架构、参数校验、日志安全、接口性能的强制标准。
-   **MySQL 性能规范**：索引、锁、分页、查询优化的防坑指南。
-   **线上故障排查**：低 Token 精准定位，快速解决内存泄漏、异步异常等线上问题。
