## 一、全局通用硬性规则（零例外）

所有编码场景、所有代码文件必须无条件遵守，无特殊场景豁免：

- **禁止输出残缺Demo代码**：不允许出现逻辑截断、未处理边界场景、遗留调试代码、仅测试可用的代码。所有输出必须通过生产工程校验，满足线上运行标准。

- **全生命周期闭环兜底**：所有手动创建的资源（定时器、事件监听、闭包缓存、网络请求实例、全局挂载变量、自定义订阅），必须配套**精准对应的销毁、清空、卸载逻辑**，杜绝常驻内存残留。

- **安全硬编码禁令**：所有敏感配置（密钥、盐值、数据库密码、服务端口、域名白名单）禁止硬编码，统一通过环境变量注入隔离，杜绝信息泄露。

- **杜绝静默失败**：所有高危逻辑（异步请求、数据转换、数据库操作、参数解析）必须配备完整异常捕获、错误兜底、异常提示，禁止无报错、无响应的隐性失败。

- **低Token精准排查机制**：严禁全量代码、全项目扫描。根据报错现象锁定最小故障代码片段，仅针对性分析、定位、修复问题，杜绝无效Token消耗和冗余分析。

- **长对话规则锁定**：多轮对话上下文稀释时，主动坚守规范标准，禁止回归AI默认模板生成逻辑，杜绝规则失效、代码降级。

- **代码风格与目录规范**：严格遵循统一命名体系，组件使用PascalCase，函数/变量使用camelCase，文件目录使用kebab\-case；禁止大小写混乱、命名随意、文件乱放。严格复用项目现有目录结构，禁止AI自行新增杂乱目录。

- **单一职责与代码行数约束**：严格禁止超大面条函数，单个纯业务函数代码行数不得超过20行；一个函数只做一件事，复杂逻辑必须拆分子函数/工具方法，杜绝堆砌式烂代码。

- **复用优先原则**：生成代码前优先检索项目现有工具、组件、公共方法、请求封装，禁止重复造轮子、禁止重复封装相同逻辑，保证项目一致性。

- **TypeScript 强制类型约束**：所有变量、函数入参、出参、接口数据必须定义明确类型/Interface；禁止滥用any、禁止类型模糊、禁止类型逃逸；复杂业务类型必须抽离单独类型文件。

- **Git 提交与代码提交规范**：所有代码提交遵循统一 commit 规范（feat新功能/fix修复/docs文档/style样式/refactor重构/test测试/chore构建）；禁止无意义提交、批量混杂提交；单次提交只处理单一业务/单一修复，保证版本日志可追溯、可回滚。

- **代码注释强制规范**：复杂业务逻辑、特殊算法、边界处理、第三方适配逻辑必须写业务注释；通用工具函数、公共方法必须写功能、入参、出参说明；禁止冗余废话注释、禁止代码不更新注释残留过期注释。

- **统一错误码与返回规范**：所有接口、异常输出必须使用项目统一封装错误码、提示文案；禁止随意手写报错文本、禁止返回格式混乱；区分客户端错误、服务端错误、权限错误、参数错误，标准化输出。

- **跨域统一规范**：业务跨域一律使用后端白名单配置，禁止前端临时跨域兜底、禁止全局允许所有域名；本地开发代理统一配置，线上禁止代理穿透，杜绝跨域安全漏洞。

- **数据缓存规范**：本地缓存（localStorage/sessionStorage）只存非敏感配置、展示类数据；Token、用户隐私数据禁止本地明文存储；缓存必须设置过期策略，禁止永久缓存脏数据，更新数据同步更新缓存。

- **浏览器与版本兼容性规范**：编写代码禁止使用废弃API、废弃语法；优先使用兼容语法，如需新特性必须做降级兜底；适配主流现代浏览器，避免线上部分机型、低版本浏览器功能失效。

## 二、JavaScript 企业级编码规范

所有 JS/TS 业务逻辑、工具函数、数据处理、异步逻辑、生命周期逻辑，必须严格遵守本生产级强制规范。统一代码风格、规避低级语法Bug、杜绝数据污染与内存泄漏、解决异步静默失败、时序错乱等线上高频问题，所有输出代码默认满足生产环境稳定部署标准。

- **一、基础语法强制规范（杜绝低级语法 Bug）**变量声明严格使用 `const / let`，完全废弃 `var`，杜绝变量提升、暂时性死区、全局变量污染、重复声明等问题。

- 所有条件判断、数值比对、等值校验统一使用 `=== / \!==`，禁止使用 `== / \!=`，杜绝隐性类型转换导致的逻辑异常。

- 所有变量必须提前显式声明，禁止未声明直接赋值，杜绝隐式挂载 window/global 造成隐秘全局污染。

- 全程使用标准 ES6\+ 语法开发，禁止使用废弃 API、废弃事件、老旧兼容写法，保持代码现代化、标准化、可维护。

- **二、全局与原型安全规范（杜绝污染冲突）**严禁修改、重写、拓展 Object、Array、String、Number、Function 等原生构造函数，彻底规避原型污染风险。

- 所有公共通用工具方法，统一独立封装专属工具模块，禁止挂载在原生原型上，避免全局逻辑冲突、功能覆盖。

- 禁止随意挂载 window / global 全局变量，全局状态统一通过状态管理或公共模块统一维护，严控全局变量泛滥。

- **三、This 上下文绑定规范（统一执行上下文）**存在动态 this 绑定的场景（DOM 事件回调、定时器业务逻辑、对象方法调用），统一使用普通函数。

- 仅需继承外层上下文、无需动态绑定 this 的场景，统一使用箭头函数，禁止无意义随意混用两种函数写法。

- **四、异步编程标准化规范（根治异步 Bug、静默失败）**精准选型 Promise 方法，禁止无脑滥用：多任务强一致性、全部成功才算成功使用 `Promise\.all`；允许部分失败、不阻断整体流程使用 `Promise\.allSettled`；需要超时拦截、竞态处理使用 `Promise\.race`。

- 无依赖的并行请求必须并行执行，仅存在前后参数依赖的逻辑允许串行 await，禁止无意义串行堆叠、叠加请求耗时。

- 大批量循环异步任务必须做并发限流，禁止无限制批量发起请求，避免浏览器请求队列堵塞、页面卡死、接口被平台限流封禁。

- 所有 Promise、async/await 异步逻辑必须配置完整 catch 异常捕获与业务兜底，杜绝异步静默失败、无日志、无报错、无提示的隐性问题。

- **五、数据处理规范（杜绝数据变异、赋值异常、渲染报错）数据拷贝规范**：单层基础数据可使用浅拷贝；嵌套对象、多级表单、树结构、可编辑回显数据必须使用深拷贝；包含函数、正则、特殊对象的复杂数据，禁止使用 JSON 序列化方式拷贝，避免数据丢失、属性截断。

- **数组操作规范**：只读遍历、映射、筛选、转换场景，统一使用 map/filter/reduce/for\.\.\.of；如需修改数组数据，必须先拷贝再操作，禁止直接变更原数组。禁止随意修改数组 length、直接修改数组下标，杜绝隐性数据变异、Vue 响应式失效、业务数据异常变动。

- **特殊数据结构规范**：Set、Map 专属数据结构必须使用对应专用遍历方式，禁止直接套用普通数组方法，防止数据遍历丢失、取值错乱。

- **空值与数值兜底规范**：多层嵌套对象、接口不确定返回数据，必须做空值判断与兜底处理；所有数值计算、除法、浮点运算，必须拦截 `NaN`、`Infinity` 非法值，杜绝页面空白、渲染报错。

- **空值合并严谨规范**：业务默认值兜底优先使用 `??` 空值合并运算符；禁止滥用 `\|\|`，避免误覆盖 0、false、\&\#39;\&\#39; 等合法业务空值，防止业务逻辑误判。

- **六、工具函数与通用逻辑规范（高复用、高容错、低冗余）**严格遵循单一职责原则，禁止编写超大面条函数，复杂业务逻辑必须拆分为独立子函数、工具方法，单个函数仅处理单一业务逻辑。

- 所有公共工具函数、通用业务方法，入参必须配置默认值与容错处理，不依赖外部传参的完整性，杜绝参数缺失导致的代码报错。

- 防抖、节流工具函数必须内置 cancel 重置、销毁能力；组件内注册的高频事件防抖节流逻辑，必须在组件卸载阶段清空重置，避免残留执行、内存堆积。

- **七、内存与闭包生命周期规范（根治内存泄漏、僵尸逻辑）**禁止多层闭包嵌套、禁止闭包嵌套定时器/异步逻辑，规避变量捕获错乱、偶现时序 Bug、长期内存滞留问题。

- 所有定时器、延时器、原生事件监听、异步 pending 请求、自定义订阅监听，必须配套完整的清空、解绑、销毁逻辑，实现生命周期完全闭环，无残留僵尸逻辑。

- **八、业务统一规范（全局风格统一、可维护性拉满）**禁止代码内散落自定义时间戳、零散日期格式、手动时间计算逻辑，全局时间运算、时间格式化统一使用项目公共工具方法，保证全局格式统一、逻辑统一。

- 复杂业务逻辑、边界判断、特殊兼容处理、自定义算法逻辑，必须编写清晰业务注释；及时同步更新注释，禁止残留过期、无效、冗余废话注释。

## 三、Vue3 生产级强制规范

- **一、响应式核心强制规范（杜绝响应式失效）**基础原始类型（字符串、数字、布尔）统一使用 `ref`，复杂对象、嵌套数组统一使用`reactive`，禁止交叉混用、场景错配。

- 禁止直接解构 `reactive` 原始对象，解构必须搭配 `toRefs`，完整保留组件响应式能力。

- 禁止整体赋值替换 `reactive` 响应式对象，会直接摧毁响应式链路；数据更新统一使用`Object\.assign` 或逐字段增量修改。

- **二、渲染与列表性能规范（杜绝卡顿、渲染错乱）**严格区分渲染指令场景：高频切换组件（弹窗、下拉、悬浮提示）用 `v\-show`；低频展示、权限控制、需销毁重建的组件用 `v\-if`，禁止场景颠倒。

- 列表渲染严禁使用数组索引 index 作为 key，必须使用后端返回唯一业务ID，彻底规避 DOM Diff 节点复用错乱、数据渲染重叠问题。

- 数据量 ≥ 1000 条的长列表，必须启用虚拟列表、分页懒加载、滚动节流优化，禁止一次性全量渲染导致页面卡顿、内存飙升。

- **三、组件通信标准化规范（统一组件交互）**Props 禁止数组简写模式，必须完整配置：类型校验、是否必填、默认值、自定义校验规则，杜绝外部传参不规范导致页面隐性异常。

- 组件所有自定义派发事件，必须通过 `emits` 显式声明，禁止隐式随意抛事件，保证组件通信可追溯、可维护、无冗余。

- **四、计算属性与侦听器规范（纯净无副作用）**Computed 强制为纯函数，只读、无副作用；内部禁止异步请求、禁止修改外部数据、禁止写入操作，异步派生逻辑统一使用 watch 实现。

- Watch 精准配置参数：监听嵌套对象/数组必须开启 `deep`，需要页面初始执行必须开启 `immediate`；组件内 watch 自动随组件销毁解绑，禁止手动常驻监听残留。

- **五、页面架构工程规范（高内聚低耦合）**复杂页面严格按业务模块拆分组合式函数（Composables），拆分复用逻辑，禁止所有业务逻辑堆砌在单文件组件中，避免代码臃肿、难以维护。

- **六、全生命周期闭环规范（含KeepAlive缓存专属方案，根治内存泄漏）通用生命周期硬性规则**：组件挂载阶段创建的所有资源，卸载阶段必须100%销毁清零，杜绝僵尸逻辑、后台残留、内存泄漏。

- 定时器/延时器必须保存实例ID，普通组件在 `onUnmounted` 统一清空；所有 pending 态网络请求，组件卸载必须主动取消，禁止异步回调继续修改已销毁组件状态。

- 所有原生事件监听、自定义事件订阅、全局事件挂载，组件卸载必须对应解绑移除，杜绝事件堆积、重复触发。

- 轮询、心跳、定时刷新、递归请求等长效逻辑，必须配置销毁开关，组件退出立即终止循环执行。

- 图表、地图、富文本、播放器等第三方实例，组件卸载必须主动调用 `destroy/unmount` 销毁方法，释放堆内存资源。

- 禁止在组件卸载、DOM销毁后，执行DOM查询、DOM修改、状态更新、弹窗提示等无效报错逻辑。

- **KeepAlive 缓存组件专属兜底规则（核心生产级规则）**禁止仅依靠 `onUnmounted` 清理资源，缓存组件不会触发该钩子，会永久残留内存泄漏。

- 缓存组件资源清理采用**双销毁策略**：定时器、请求、监听、副作用，同时在 `onDeactivated \+ onUnmounted` 执行清理。

- 组件被缓存失活（onDeactivated）必须暂停轮询、心跳、实时订阅等后台持续逻辑，禁止静默消耗资源。

- 缓存组件重新激活（onActivated）必须重置加载状态、清空过期请求上下文、刷新旧缓存数据，避免页面展示旧逻辑、旧状态。

- 高频运行定时、实时订阅、视频播放的动态组件，禁止加入 KeepAlive 缓存白名单，规避长期后台运行风险。

## 四、工程化、打包、部署生产规范

- **环境严格隔离**：开发环境、生产环境配置完全独立，禁止配置混用、一刀切通用配置。

- **生产环境强制开启**：代码压缩、Tree\-shaking、Gzip压缩、路由懒加载、第三方依赖分包拆分、静态资源CDN映射。

## 六、MySQL/SQL 安全与性能规范
